# Specifications: Frontend Component and API Contracts

## API Interoperability

All requests to the backend must follow these specifications:
- **Base URL**: `http://localhost:8000`
- **Headers**:
    - `Content-Type: application/json`
    - `Authorization: Bearer <token>` (for protected routes)

### Endpoints Detail

| Feature | Method | Endpoint | Description |
| --- | --- | --- | --- |
| Health | `GET` | `/health` | Verify connectivity |
| Login | `POST` | `/auth/login` | Returns `{access_token, token_type}` |
| Get Patients | `GET` | `/pacientes` | Returns list of patient profiles |
| Get Alerts | `GET` | `/alertas` | Returns real-time triage alerts |
| Get Telemetry | `GET` | `/telemetria` | Returns biometric data streams |
| Get Triage | `GET` | `/triajes` | Returns evaluation history |

## Component Specifications

### 1. LoginComponent
- **Inputs**: Email (string), Password (string).
- **Validation**: Required fields.
- **Output**: On success, navigate to `/dashboard`. On failure, show error message.

### 2. DashboardComponent
- **Metrics Grid**:
    - `Total Patients`: Count of active patients.
    - `Critical Alerts`: Count of high-risk alerts in the last 24h.
    - `Open Triage`: Count of pending evaluations.
- **Alert List**: Dynamic card feed showing patient name, risk level, and timestamp. Ids should be clickable to view patient telemetry.

### 3. PatientListComponent
- **Filter**: Filter by Risk Level (High, Medium, Low).
- **Table Columns**: Name, Age, Identification, Last Triage Status.

### 4. TelemetryView (Detail)
- **Chart**: X-axis (time), Y-axis (Value). Support multiple series (Heart Rate, Respiratory Rate).
- **AI Explanation Section**: Display text generated by the Predictive Model / Agentic AI.

## Security Requirements
- **JWT Storage**: Store token in `localStorage`.
- **Session Expiry**: Clear `localStorage` and redirect to `/login` if a `401 Unauthorized` is received.
